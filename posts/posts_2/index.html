SSH service

Why?

There are two main reasons for me to start SSH service on WSL:

  1. It’s convenient for me to operate my host machine in the lab.
  2. As long as there is a low-end, lightweight MacBook, we can deal with some tasks with SSH connecting to my host machine

How?

I tried many tutorials on the Internet, most of them don’t work. So, I want to record how to do it step by step (Reference blog).

  1. Guest machine and local machine need to be the same local area network. (If not, it is still possible to manipulate the host through internal network penetration, probably I’ll post it later.)
  2. Reinstall the openssh-server 
    apt remove openssh-server
apt install openssh-server
  3. Edit the /etc/ssh/sshd_config (change the Port from 22 to 2222) 
    vi /etc/ssh/sshd_config
  4. Start the ssh service 
    service ssh restart
  5. Open the window powershell, tab the following commands 
    netsh interface portproxy add v4tov4 listenaddress=0.0.0.0 listenport=2222 connectaddress=[WSL2_IP] connectport=[PORT]
netsh advfirewall firewall add rule name=WSL2 dir=in action=allow protocol=TCP localport=2222
    (we can use ifconfig in WSL2 to get [WSL2_IP], and we can ask chatGPT for details.) (we can get [Port] from the /etc/ssh/sshd_config)
  6. Log in the account by SSH 
    ssh user_name@[Window_IP] -p [Port]
    (we can use ipconfig in window powershell to get [windows_IP], ask we can ask chatGPT for details.) (we can get [Port] from the /etc/ssh/sshd_config)
