#Aspera

Why use aspera:

a. Reliability and integrity (ensures complete data integrity with checks at every packet transfer point and automatic retry mechanisms that handle intermittent network issues without any data loss or corruption.); b. High-speed transfers

Installation on linux:

We can use conda or source code to install. Here we choose conda to install (reference link)

An example of using aspera to download fastq data

When I studied the docker-based Hi-C pipeline CustardPy, I needed to download a series of fastq data: SRR17870713, SRR17870718, SRR17870740.

  1. Enter SRR17870713 in ENA; Then we can click Study Accession to get all data related to SRR17870713;

  2. We can download the TSV to extract the ‘fastq_aspera’ column which contains all the data in the same study, and save it as id.txt

  3. Finally, we can use the following shell script (provided by Prof.Jiankang Wang) to download all the data using aspera

    key=/home/yinpf/.aspera/connect/etc/asperaweb_id_dsa.openssh
for round in $(seq 1 10)
do
     for i in `cat id.txt`
     do
             echo $i
             shorti=$(echo "$i" | sed 's|ftp.sra.ebi.ac.uk||')
             ascp -QT -k 1 -v -l 300m -P33001 -i $key era-fasp@fasp.sra.ebi.ac.uk:$shorti .
     done
done

  4. Additionally, we can also use SRA Explorer to download these data. Enter SRR17870713, SRR17870718, SRR17870740 in
    (Written with the help of Prof.Jiankang Wang and Yongwen Ding)
